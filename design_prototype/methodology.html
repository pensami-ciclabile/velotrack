Scientific Framework

The Methodology of Urban Movement

A technical overview of how we process millions of data points from Milan's tram network to create real-time transit intelligence.

database

01. Data Collection

Continuous ingestion of real-time GTFS-RT feeds from ATM Milan. We aggregate positioning data from over 250 active tram cars every 3 seconds, ensuring sub-meter accuracy in our spatial database.

DB
PostgreSQL / PostGIS Stack
location_on

02. GPS Parsing

Filtering noise from urban canyons. Our custom Kalman Filter algorithm accounts for signal multipath effects near the Duomo and high-density Brera streets.

99.2% Accuracy Threshold

departure_board

03. Stop Detection

Automated dwell-time analysis at each platform. We distinguish between traffic-related stops and passenger boarding.

Dwell Variable μ = 24.2s
analytics

04. Network Analytics

Applying graph theory to identify bottleneck nodes. By modeling the tram network as a directed weighted graph, we predict system-wide delays before they propagate to outer rings.
